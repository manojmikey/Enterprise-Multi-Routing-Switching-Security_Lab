# OSPF — Open Shortest Path First

## What Is OSPF?

OSPF is a **link-state routing protocol** used inside a single autonomous system (AS). It is one of the most widely deployed interior gateway protocols (IGPs) in enterprise networks.

Unlike distance-vector protocols (like RIP) that just share routing tables with neighbors, OSPF routers share information about the **entire topology** of the network. Every router builds a complete map of the network and calculates the best path independently using Dijkstra's Shortest Path First (SPF) algorithm — hence the name.

---

## Why Use OSPF Instead of Static Routes?

| Static Routes | OSPF |
|---|---|
| You manually configure every path | Routers discover paths automatically |
| A link failure means manual update | Routers reconverge automatically |
| Doesn't scale in large networks | Scales well with areas |
| No loop prevention built in | Loop-free by design (SPF algorithm) |

In an enterprise, you can't manually manage routes across dozens of routers. OSPF handles it dynamically.

## Key OSPF Concepts

### 1. Router ID
Every OSPF router needs a unique identifier — this is the **Router ID (RID)**. It's a 32-bit number written in dotted decimal format (like an IP address).

Best practice: explicitly set the Router ID to a loopback IP so it never changes even if physical interfaces go down.

```
router ospf 1
  router-id 1.1.1.1
```

### 2. Areas
OSPF uses **areas** to scale. An area is a logical grouping of routers and links. All areas must connect to **Area 0** (the backbone area).

In this lab, all OSPF routers are in **Area 0** — this is the simplest single-area OSPF design, typical in smaller enterprise cores.

### 3. OSPF Neighbor Adjacency
Before exchanging routes, OSPF routers go through a **neighbor adjacency** process:

```
Down → Init → 2-Way → ExStart → Exchange → Loading → Full
```

- **Down** — No hellos received yet
- **Init** — Hello received but our Router ID not in it yet
- **2-Way** — Both routers see each other (DR/BDR election happens here on broadcast networks)
- **ExStart / Exchange** — Routers exchange Database Description (DBD) packets to compare LSDBs
- **Loading** — Requesting missing LSAs
- **Full** — Complete, synchronized topology databases ✅

The goal is to reach **Full** state. On broadcast networks (like Ethernet), you'll see **Full/DR** or **Full/BDR** or **Full/DROTHER** in the neighbor table.

### 4. Hello Protocol
OSPF routers discover each other by sending **Hello packets** at regular intervals. For OSPF to form a neighbor relationship, both sides must agree on:

- Hello interval (default 10s on Ethernet)
- Dead interval (default 40s on Ethernet)
- Area ID
- Subnet mask
- Authentication (if configured)

If any of these mismatch, neighbors won't form.

### 5. Link-State Advertisements (LSAs)
OSPF routers flood **LSAs** throughout their area to describe the topology. Each router stores all received LSAs in its **Link-State Database (LSDB)**.

Common LSA types:

| Type | Name | Purpose |
|---|---|---|
| Type 1 | Router LSA | Describes a router's links |
| Type 2 | Network LSA | Describes a broadcast segment (generated by DR) |
| Type 3 | Summary LSA | Routes between areas (generated by ABR) |
| Type 5 | External LSA | Routes from outside OSPF (redistributed routes) |

### 6. DR and BDR Election
On broadcast networks (Ethernet), OSPF elects a **Designated Router (DR)** and **Backup Designated Router (BDR)** to reduce the number of LSA floods. Instead of every router flooding to every other router (O(n²) problem), all routers only send LSAs to the DR/BDR.

Election is based on:
1. Highest OSPF **priority** (default 1; set to 0 to exclude from election)
2. Tiebreak: Highest **Router ID**

### 7. SPF Algorithm
Once every router has the same LSDB, each router independently runs **Dijkstra's SPF algorithm** to compute the shortest (lowest cost) path to every destination. OSPF cost = **100 / interface bandwidth (in bps)**. A 100 Mbps link has a cost of 1; a 10 Mbps link has a cost of 10.

## How OSPF Was Used in This Lab

OSPF was configured as the **internal routing protocol** for the HQ network. R1, R3, and R4 all participate in OSPF Area 0.

The purpose: R1 needed to dynamically know about the networks reachable via R3 and R4 without manually adding static routes. OSPF handles this automatically.

Additionally, BGP routes were **redistributed into OSPF** to give internal OSPF routers reachability to external networks (like R5's loopback 5.5.5.5).

---

## Configuration Applied

### R1
```
router ospf 1
 router-id 1.1.1.1
 network 11.11.13.0 0.0.0.255 area 0
 network 11.11.14.0 0.0.0.255 area 0
 network 192.168.12.0 0.0.0.255 area 0
```

### R3
```
router ospf 1
 router-id 3.3.3.3
 network 11.11.13.0 0.0.0.255 area 0
```

### R4
```
router ospf 1
 router-id 4.4.4.4
 network 11.11.14.0 0.0.0.255 area 0
```

### BGP → OSPF Redistribution (on R1)
This was needed so that OSPF routers could reach BGP-learned external networks (e.g., R5's loopback):
```
router ospf 1
 redistribute bgp 100 subnets
```

> ⚠️ **Note:** `subnets` keyword is required when redistributing into OSPF — without it, only classful networks get redistributed.

```
show ip ospf database           → View the LSDB
show ip route ospf              → View only OSPF-learned routes (marked with 'O')
show ip ospf interface brief    → Check interfaces running OSPF
```

## Common OSPF Issues

| Problem | Likely Cause | Fix |
|---|---|---|
| Neighbors stuck in INIT | One side not receiving Hellos | Check interface, subnet, duplex |
| Neighbors stuck in EXSTART/EXCHANGE | MTU mismatch | Match MTU or use `ip ospf mtu-ignore` |
| Missing routes | `network` statement wrong | Check wildcard mask |
| Routes not redistributed | Missing `subnets` keyword | Add `subnets` to redistribute command |
